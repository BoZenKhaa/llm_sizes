## Workflow for each new lab

See `CLAUDE.md` for the authoritative file layout + merge commands. Summary:

1. Orchestrator creates `labs/<lab>.csv` (header copied from main CSV) and `labs/<lab>.notes.md` (heading only).
2. Dispatch `researcher` subagent scoped to the single lab, pointed at `labs/<lab>.*`. Multiple researchers may run in parallel since each owns its own files.
3. Dispatch `verifier` subagent(s) on the lab CSV (batch by 10 for parallelism). Verifier first runs a schema-conformance diff against the main CSV header.
4. If verifier reports hard FAILs, dispatch `researcher` in fixup mode on the same lab file with the exact FAIL list.
5. Spot-check any row whose cited URL looks unfamiliar — the Mythos pass showed that "source text in `param_source_note` that isn't in the two URL columns" is the main hallucination failure mode.
6. Orchestrator merges the lab CSV + notes into the main files (`tail -n +2 labs/<lab>.csv >> llm_sizes.csv` + `cat labs/<lab>.notes.md >> RESEARCH_NOTES.md`). Keep the lab file as provenance.
7. Check the lab off here and commit.
